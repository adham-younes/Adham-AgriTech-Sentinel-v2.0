# تنفيذ التحقق والتشغيل ورفع التقرير الفني

## نطاق التنفيذ

* إعداد البيئة، تشغيل فحوصات الساكنة والاختبارات، تشغيل الخادم، فحص نقاط الصحة وواجهات API، وتوليد تقرير نتائج عملي.

## خطوات التنفيذ

1. إعداد بيئة التشغيل:

   * نسخ `.env.example` إلى `.env` وملء المفاتيح المطلوبة (`Supabase`, `OpenWeather`, `Mapbox`, `EOSDA`, `OpenAI/Groq`).

   * تثبيت الاعتمادات: `npm ci`.
2. فحوصات الساكنة والاختبارات:

   * تشغيل `npm run type-check` و`npm run lint`.

   * تشغيل `npm test`؛ مراجعة نتائج `__tests__/api/*`.
3. تشغيل خادم التطوير والتحقق:

   * تشغيل `npm run dev`.

   * التحقق من الواجهة (`app/page.tsx`, `app/dashboard/*`).

   * فحص `/api/services/health` للتكامل مع الخدمات.
4. اختبار الواجهات الخارجية:

   * تشغيل `scripts/test-apis.js` بعد ضبط المفاتيح؛ مراقبة الحدود والتعامل مع الأخطاء.
5. مراجعة الأمن والسجلات:

   * تدقيق سياسات RLS في `scripts/013_supabase_schema.sql`.

   * مراجعة التسجيل واستجابات الأخطاء في `app/api/*`؛ اقتراح مُسجّل موحّد.
6. إعدادات البناء والنشر:

   * مراجعة `next.config.mjs`, `vercel.json`, إعداد الصور والرؤوس الأمنية وإعادة الكتابة.

   * مراجعة إعدادات Docker وCI (`docker-compose.ci.yml`, `\.github/workflows/ci.yml`).

## مخرجات متوقعة

* تقرير نتائج تشغيل وفحوصات واختبارات مع توصيات عملية قصيرة بالأولوية.

* قائمة تحسينات سريعة جاهزة للتطبيق ضمن جلسة لاحقة.

## افتراضات

* توفر مفاتيح الخدمات اللازمة للتشغيل والاختبار.
